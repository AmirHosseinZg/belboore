# Definition of Done

A task is done only if:

## Functional
- feature works as intended
- edge cases are handled reasonably
- mobile and desktop both function

## UX
- interface looks polished
- primary actions are obvious
- states are not confusing

## SEO
- metadata exists
- page has useful copy
- internal links exist where relevant
- headings are meaningful

## I18N
- Persian and English variants both exist
- route and directionality are correct
- metadata localized

## Code Quality
- reusable where appropriate
- naming is clear
- no unnecessary complexity
- no dead code introduced

## Documentation
- relevant docs updated if decisions changed

## Deployability
- builds successfully
- no runtime dependency surprises
