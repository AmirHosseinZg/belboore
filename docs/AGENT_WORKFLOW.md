# Agent Workflow

## Workflow Mode
Single-agent, docs-first, low-assumption execution.

## Process
1. Read source-of-truth docs
2. Confirm scope
3. State assumptions
4. Update docs if assumption becomes a decision
5. Implement in small batches
6. Verify against acceptance criteria

## Task Output Format
- Objective
- Inputs used
- Assumptions
- Files touched
- Implementation summary
- Validation checklist

## Rules
- no silent assumptions
- no backend overreach
- no dependency sprawl
- no thin SEO pages
- no placeholder-looking homepage

## Priority Order
1. SEO foundation
2. information architecture
3. homepage UX
4. built-in tool usability
5. polish
