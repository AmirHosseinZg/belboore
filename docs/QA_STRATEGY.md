# QA Strategy

## Areas to Validate
1. Routing
2. i18n behavior
3. theme behavior
4. homepage layout
5. category navigation
6. tool page rendering
7. built-in tool usability
8. redirect behavior
9. metadata output
10. responsiveness
11. accessibility basics

## Built-In Tool QA
### Image tools
- handles valid images
- shows clear output options
- no confusing failure states

### Text tools
- text input, transform, copy
- empty state behavior

### JSON tool
- valid JSON formatting
- invalid JSON error state

## Search QA
- finds tools by English and Persian names
- handles tags and synonyms
- works on mobile

## SEO QA
- title tags
- meta descriptions
- canonicals
- hreflang
- OG tags
- schema presence where intended

## Accessibility QA
- keyboard navigation
- visible focus
- form labels
- contrast sanity
